## 0. Preamble

### 0.1 Purpose

This document defines the complete formal specification for Sunny, a music theory engine whose objective is the computational formalisation of Western tonal, post-tonal, and transformational music theory. The specification is structured so that every well-formed construct in these theoretical systems is representable, derivable, and manipulable within its framework.

The specification is designed for implementation in a systems language (C++23) with exact arithmetic in pitch and rhythm domains, result-type error propagation, and real-time safety constraints in audio-path code.

### 0.2 Scope

The primary working substrate is 12-TET. Foundational abstractions are parameterised over a temperament division count *n* so that extension to arbitrary *n*-EDO, just intonation, and historical temperaments requires instantiation of existing type parameters rather than structural modification.

### 0.3 Notation Conventions

Throughout this specification:

- **Z** denotes the integers; **Q** the rationals; **R** the reals; **R+** the positive reals.
- **Z/nZ** denotes the cyclic group of integers modulo *n*, with elements {0, 1, ..., n вҲ’ 1} and addition modulo *n*.
- Pitch classes in 12-TET are elements of **Z/12Z**.
- Functions are written *f* : *A* вҶ’ *B*. Partial functions are written *f* : *A* вҮҖ *B*.
- Group actions are written as left actions: *g* В· *x* for group element *g* acting on *x*.
- Interval vectors are written as ordered tuples in angle brackets: вҹЁ*a*вӮҒ, *a*вӮӮ, ..., *a*вӮҶвҹ©.
- The symbol вүЎ denotes equivalence modulo a specified relation.
- **Beat**(*p*, *q*) denotes an exact rational value *p*/*q* where *p* вҲҲ **Z**, *q* вҲҲ **Z+**.
- Error-returning operations use the notation *f* : *A* вҶ’ *B* | *E*, corresponding to `std::expected<B, E>` in C++23.

### 0.4 Computability Classification

Each definition and operation in this specification is annotated with one of three categories:

- **[C]** вҖ” *Computable*: Algorithmically derivable from primitives with no ambiguity. Implementations must produce identical results for identical inputs.
- **[H]** вҖ” *Heuristic*: Requires stylistic, aesthetic, or contextual weighting. The specification defines the constraint space; an implementation selects within it according to configurable policy.
- **[E]** вҖ” *Empirical*: Depends on perceptual data, corpus statistics, or experimental measurement. The specification defines the interface; data is injected at runtime.

### 0.5 Cross-Reference to Existing Implementation

Where Sunny already implements a construct, the specification notes the current coverage and any gaps requiring amendment. Constructs absent from the current implementation are marked with **[NEW]**.

## 1. Algebraic Foundations

### 1.1 The Cyclic Group Z/*n*Z

**Definition 1.1.1** [C]. Let *n* вҲҲ **Z+** be the temperament division count. The *pitch class group* is the cyclic group (**Z**/*n***Z**, +), whose elements are the residue classes {0, 1, ..., *n* вҲ’ 1} under addition modulo *n*.

For 12-TET, *n* = 12. The group has the following properties:

- **Closure**: For all *a*, *b* вҲҲ **Z**/*n***Z**, (*a* + *b*) mod *n* вҲҲ **Z**/*n***Z**.
- **Associativity**: (*a* + *b*) + *c* вүЎ *a* + (*b* + *c*) (mod *n*).
- **Identity**: 0 is the identity element; *a* + 0 вүЎ *a* (mod *n*).
- **Inverse**: For each *a*, the inverse is (*n* вҲ’ *a*) mod *n*.
- **Commutativity**: *a* + *b* вүЎ *b* + *a* (mod *n*).

**Status**: Implemented. Current code realises transposition as addition in **Z/12Z** and inversion as negation.

### 1.2 The Dihedral Group D*n*

**Definition 1.2.1** [C]. The *dihedral group* D*n* of order 2*n* is generated by transposition *T* and inversion *I*, subject to the relations:

- *T*вҒҝ = *e* (identity)
- *I*ВІ = *e*
- *I* В· *T* = *T*вҒ»В№ В· *I*

Its elements are {*T*вӮҖ, *T*вӮҒ, ..., *T*вӮҷвӮӢвӮҒ, *I*вӮҖ, *I*вӮҒ, ..., *I*вӮҷвӮӢвӮҒ}, where:

- *T*вӮ–(*x*) = (*x* + *k*) mod *n*  (transposition by *k*)
- *I*вӮ–(*x*) = (*k* вҲ’ *x*) mod *n*  (inversion about axis *k*/2)

For 12-TET, DвӮҒвӮӮ has order 24 and acts on **Z/12Z**.

**Composition law**: For all *T*вӮҗ, *T*бөҰ, *I*вӮҗ, *I*бөҰ:

- *T*вӮҗ вҲҳ *T*бөҰ = *T*вӮҚвӮҗвӮҠбөҰвӮҺ вӮҳвӮ’рқ’№ вӮҷ
- *I*вӮҗ вҲҳ *T*бөҰ = *I*вӮҚвӮҗвӮӢбөҰвӮҺ вӮҳвӮ’рқ’№ вӮҷ
- *T*вӮҗ вҲҳ *I*бөҰ = *I*вӮҚвӮҗвӮҠбөҰвӮҺ вӮҳвӮ’рқ’№ вӮҷ
- *I*вӮҗ вҲҳ *I*бөҰ = *T*вӮҚвӮҗвӮӢбөҰвӮҺ вӮҳвӮ’рқ’№ вӮҷ

**Status**: Complete. PTDN001A: DвӮҒвӮӮ composition, inverse, apply, order, conjugacy classes (9), subgroup generation, named subgroups (ZвӮҒвҖ“ZвӮҒвӮӮ, DвӮҒвҖ“DвӮҒвӮӮ).

### 1.3 The Free Abelian Group of Intervals

**Definition 1.3.1** [C]. The *interval group* is the free abelian group **Z** Г— **Z**, whose elements (*c*, *d*) represent intervals with chromatic component *c* (semitone displacement) and diatonic component *d* (letter-name displacement).

- **Addition**: (*c*вӮҒ, *d*вӮҒ) + (*c*вӮӮ, *d*вӮӮ) = (*c*вӮҒ + *c*вӮӮ, *d*вӮҒ + *d*вӮӮ)
- **Identity**: (0, 0), the unison.
- **Inverse**: вҲ’(*c*, *d*) = (вҲ’*c*, вҲ’*d*), the interval measured in the opposite direction.

This group is not cyclic; it has rank 2. The chromatic component alone projects onto **Z** (or **Z/12Z** for pitch classes); the diatonic component projects onto **Z/7Z** for diatonic contexts.

**Status**: Implemented. PTDI001A: DiatonicInterval as (chromatic, diatonic) pair with quality derivation, interval addition, inversion, and application on SpelledPitch.

### 1.4 Group Actions

**Definition 1.4.1** [C]. A *group action* of a group (*G*, В·) on a set *S* is a function ПҶ : *G* Г— *S* вҶ’ *S* satisfying:

- ПҶ(*e*, *s*) = *s* for all *s* вҲҲ *S*
- ПҶ(*g*вӮҒ, ПҶ(*g*вӮӮ, *s*)) = ПҶ(*g*вӮҒ В· *g*вӮӮ, *s*) for all *g*вӮҒ, *g*вӮӮ вҲҲ *G*, *s* вҲҲ *S*

The following group actions are foundational to the specification:

| Group | Set | Action | Domain |
|-------|-----|--------|--------|
| **Z/12Z** | Pitch classes | Transposition | В§2, В§3 |
| DвӮҒвӮӮ | Pitch classes | Transposition + inversion | В§2, В§12 |
| DвӮҒвӮӮ | Pitch class sets | Setwise T/I | В§12 |
| **Z** Г— **Z** | Pitches (with spelling) | Interval application | В§3 |

| SвӮҒвӮӮ | Tone rows | Permutation | В§12 |

### 1.5 Equivalence Relations

**Definition 1.5.1** [C]. The following equivalence relations are used throughout:

1. **Octave equivalence**: Pitches *p*вӮҒ вҲј *p*вӮӮ iff *p*вӮҒ вҲ’ *p*вӮӮ вүЎ 0 (mod 12). The quotient space is the set of pitch classes.
2. **Transpositional equivalence** (T-equivalence): Pitch class sets *A* вҲј_T *B* iff there exists *k* such that *T*вӮ–(*A*) = *B*.
3. **Transpositional-inversional equivalence** (TI-equivalence): *A* вҲј_{TI} *B* iff there exists *g* вҲҲ DвӮҒвӮӮ such that *g*(*A*) = *B*.
4. **Enharmonic equivalence**: Spelled pitches *p*вӮҒ вҲј_enh *p*вӮӮ iff they map to the same element of **Z/12Z**.

**Status**: Implemented. Octave equivalence, T-equivalence, and TI-equivalence via PTPS001A. Enharmonic equivalence via PTSP001A (SpelledPitch with pitch class realisation).

## 2. Pitch Space

### 2.1 Frequency Space

**Definition 2.1.1** [C]. *Frequency space* is the multiplicative group (**R+**, В·). A pitch is a point in frequency space, measured in hertz (Hz).

**Definition 2.1.2** [C]. The *concert pitch standard* is *A*вӮ„ = 440.0 Hz. This is configurable; the specification treats it as a parameter *f*_ref with default value 440.0.

### 2.2 Continuous Pitch Space

**Definition 2.2.1** [C]. *Continuous pitch space* is (**R**, +), where the value represents semitones above a reference point. The isomorphism between frequency space and continuous pitch space is:

- **Forward** (frequency to pitch): *p* = 12 В· logвӮӮ(*f* / *f*_ref) + 69
- **Inverse** (pitch to frequency): *f* = *f*_ref В· 2^((*p* вҲ’ 69) / 12)

where *p* is measured in MIDI pitch units (MIDI note 69 = A4 = *f*_ref).

This logarithmic mapping converts the multiplicative structure of frequency space to the additive structure of pitch space: multiplication of frequencies corresponds to addition of pitch values.

**Invariant**: The forward and inverse mappings are exact inverses over **R+** вҶ’ **R**. In implementation, floating-point representation introduces bounded error; for pitch-to-frequency-to-pitch round trips, the specification requires error < 2вҒ»вҒҙвҒ° semitones (sub-perceptual).

### 2.3 MIDI Pitch Space

**Definition 2.3.1** [C]. *MIDI pitch space* is the subset {0, 1, ..., 127} вҠӮ **Z**, where each integer corresponds to a semitone in 12-TET. The mapping from MIDI number *m* to frequency is:

*f*(*m*) = *f*_ref В· 2^((*m* вҲ’ 69) / 12)

**Decomposition**: Every MIDI pitch *m* decomposes uniquely as:

- *pitch class*(*m*) = *m* mod 12 вҲҲ **Z/12Z**
- *octave*(*m*) = вҢҠ*m* / 12вҢӢ вҲ’ 1 вҲҲ **Z**

so that *m* = 12 В· (*octave* + 1) + *pitch class*.

The octave convention follows scientific pitch notation: C4 = MIDI 60, so *octave*(60) = 4, *pitch class*(60) = 0.

**Status**: Implemented. MIDI decomposition, frequency conversion, and pitch class extraction are all present.

### 2.4 Pitch Class Space

**Definition 2.4.1** [C]. *Pitch class space* is **Z**/*n***Z** (Definition 1.1.1). For 12-TET:

| Pitch Class | Integer | Common Names |
|-------------|---------|--------------|
| C | 0 | BвҷҜ, Dрқ„« |
| CвҷҜ / Dвҷӯ | 1 | Bрқ„Ә |
| D | 2 | Cрқ„Ә, Eрқ„« |
| Eвҷӯ / DвҷҜ | 3 | Fрқ„« |
| E | 4 | Fвҷӯ, Dрқ„Ә |
| F | 5 | EвҷҜ, Gрқ„« |
| FвҷҜ / Gвҷӯ | 6 | Eрқ„Ә |
| G | 7 | Fрқ„Ә, Aрқ„« |
| Aвҷӯ / GвҷҜ | 8 | вҖ” |
| A | 9 | Gрқ„Ә, Bрқ„« |
| Bвҷӯ / AвҷҜ | 10 | Cрқ„« |
| B | 11 | Cвҷӯ, Aрқ„Ә |

The mapping from pitch class integer to common name is *not* a function; it is a relation (one-to-many). Disambiguation requires the *spelled pitch* representation (В§2.5).

**Status**: Implemented as integer representation. The common name table is available; context-sensitive disambiguation is not.

### 2.5 Spelled Pitch Space

**Definition 2.5.1** [C]. A *spelled pitch* is a triple (*l*, *a*, *o*) where:

- *l* вҲҲ {C, D, E, F, G, A, B} вү… **Z/7Z** is the *letter name* (diatonic pitch class)
- *a* вҲҲ **Z** is the *accidental* (number of sharps if positive, flats if negative; 0 = natural)
- *o* вҲҲ **Z** is the *octave*

**Definition 2.5.2** [C]. The *diatonic pitch class function* maps letter names to their natural pitch classes:

*nat* : **Z/7Z** вҶ’ **Z/12Z**, defined by *nat*(C) = 0, *nat*(D) = 2, *nat*(E) = 4, *nat*(F) = 5, *nat*(G) = 7, *nat*(A) = 9, *nat*(B) = 11.

**Definition 2.5.3** [C]. The *pitch class realisation* of a spelled pitch (*l*, *a*, *o*) is:

*pc*(*l*, *a*, *o*) = (*nat*(*l*) + *a*) mod 12

The *absolute pitch realisation* (MIDI number) is:

*midi*(*l*, *a*, *o*) = 12 В· (*o* + 1) + *nat*(*l*) + *a*

**Enharmonic equivalence**: Spelled pitches (*l*вӮҒ, *a*вӮҒ, *o*вӮҒ) and (*l*вӮӮ, *a*вӮӮ, *o*вӮӮ) are enharmonically equivalent iff *midi*(*l*вӮҒ, *a*вӮҒ, *o*вӮҒ) = *midi*(*l*вӮӮ, *a*вӮӮ, *o*вӮӮ).

**Pitch spelling problem** [H]: Given a pitch class *p* вҲҲ **Z/12Z** and a harmonic or melodic context, determine the most appropriate spelled pitch. This is a heuristic problem; multiple algorithms exist (Meredith's PS13, the line-of-fifths model, context-dependent rules). The specification requires:

- A default spelling that prefers sharps in sharp keys and flats in flat keys.
- An override mechanism for explicit enharmonic selection.
- A context-sensitive algorithm selectable by policy (PS13, line-of-fifths, or rule-based).

**Status**: Implemented. PTSP001A: SpelledPitch (letter, accidental, octave), SPN parsing, line-of-fifths mapping, enharmonic equivalence.

### 2.6 The Line of Fifths

**Definition 2.6.1** [C]. The *line of fifths* is the infinite sequence of spelled pitch classes generated by successive perfect fifths:

..., Gрқ„«, Dрқ„«, Aрқ„«, Eрқ„«, Bрқ„«, Fвҷӯ, Cвҷӯ, Gвҷӯ, Dвҷӯ, Aвҷӯ, Eвҷӯ, Bвҷӯ, F, C, G, D, A, E, B, FвҷҜ, CвҷҜ, GвҷҜ, DвҷҜ, AвҷҜ, EвҷҜ, BвҷҜ, Fрқ„Ә, ...

Each position on the line is indexed by an integer *q* вҲҲ **Z**, with C at position 0. The mapping from line-of-fifths position to spelled pitch class is:

- *letter*(*q*) = the letter name at position *q* mod 7 in the cycle FвҖ“CвҖ“GвҖ“DвҖ“AвҖ“EвҖ“B
- *accidental*(*q*) = вҢҠ(*q* + 1) / 7вҢӢ

The line of fifths provides a one-dimensional representation of spelled pitch class space in which:

- Key signatures correspond to contiguous segments of length 7.
- Diatonic collections are windows [*q*, *q* + 6].
- The distance between two spelled pitch classes on the line of fifths is a measure of their enharmonic remoteness.

**Status**: Implemented. PTSP001A: `line_of_fifths_position()` and `from_line_of_fifths()` constexpr functions.

### 2.7 Scientific Pitch Notation

**Definition 2.7.1** [C]. *Scientific pitch notation* (SPN) is the string representation of a spelled pitch as the concatenation of letter name, accidental symbol(s), and octave number:

SPN(*l*, *a*, *o*) = *letter*(*l*) вҖ– *accidental_str*(*a*) вҖ– *str*(*o*)

where *accidental_str*(0) = "", *accidental_str*(1) = "вҷҜ", *accidental_str*(вҲ’1) = "вҷӯ", *accidental_str*(2) = "рқ„Ә", *accidental_str*(вҲ’2) = "рқ„«", and so forth.

Parsing is the inverse: given a valid SPN string, extract (*l*, *a*, *o*).

**Status**: Implemented. PTSP001A: `from_spn()` and `to_spn()` provide first-class invertible SPN вҶ” SpelledPitch conversion.

## 3. Interval System

### 3.1 Chromatic Intervals

**Definition 3.1.1** [C]. A *chromatic interval* is an element of **Z**, representing directed semitone displacement. Positive values denote ascending intervals; negative values descending.

For pitch classes (where direction is undefined), the *interval class* is the undirected distance:

*ic*(*a*, *b*) = min((*b* вҲ’ *a*) mod 12, (*a* вҲ’ *b*) mod 12)

yielding values in {0, 1, 2, 3, 4, 5, 6}.

**Status**: Implemented.

### 3.2 Diatonic Intervals

**Definition 3.2.1** [C]. A *diatonic interval* is an element of the free abelian group **Z** Г— **Z**, written (*c*, *d*) where *c* is the chromatic component (semitones) and *d* is the diatonic component (letter-name steps, also called the *generic interval*).

The diatonic component determines the interval *number* (unison, second, third, etc.):

| *d* mod 7 | Number | Simple Name |
|-----------|--------|-------------|
| 0 | unison / octave | 1 / 8 |
| 1 | second / ninth | 2 / 9 |
| 2 | third / tenth | 3 / 10 |
| 3 | fourth / eleventh | 4 / 11 |
| 4 | fifth / twelfth | 5 / 12 |
| 5 | sixth / thirteenth | 6 / 13 |
| 6 | seventh / fourteenth | 7 / 14 |

Simple intervals have |*d*| вүӨ 6; compound intervals have |*d*| вүҘ 7.

### 3.3 Interval Quality

**Definition 3.3.1** [C]. The *quality* of a diatonic interval (*c*, *d*) is determined by comparing *c* against the *default chromatic size* for generic interval *d*. The default is the number of semitones in the corresponding diatonic scale step.

**Definition 3.3.2** [C]. The *default chromatic size function* Оҙ : **Z** вҶ’ **Z** is defined for simple intervals as:

| *d* mod 7 | Оҙ(*d*) | Category |
|-----------|--------|----------|
| 0 | 0 | Perfect |
| 1 | 2 | Major |
| 2 | 4 | Major |
| 3 | 5 | Perfect |
| 4 | 7 | Perfect |
| 5 | 9 | Major |
| 6 | 11 | Major |

For compound intervals: Оҙ(*d*) = 12 В· вҢҠ*d* / 7вҢӢ + Оҙ(*d* mod 7).

Intervals at generic positions 0, 3, 4 (unison, fourth, fifth) are in the *perfect* family. Intervals at generic positions 1, 2, 5, 6 (second, third, sixth, seventh) are in the *major/minor* family.

**Definition 3.3.3** [C]. The *quality assignment* for interval (*c*, *d*):

Let *deviation* = *c* вҲ’ Оҙ(*d*).

For *perfect*-family intervals (*d* mod 7 вҲҲ {0, 3, 4}):

| Deviation | Quality | Abbreviation |
|-----------|---------|--------------|
| ... | ... | ... |
| вҲ’2 | doubly diminished | dd |
| вҲ’1 | diminished | d |
| 0 | perfect | P |
| +1 | augmented | A |
| +2 | doubly augmented | AA |
| ... | ... | ... |

For *major/minor*-family intervals (*d* mod 7 вҲҲ {1, 2, 5, 6}):

| Deviation | Quality | Abbreviation |
|-----------|---------|--------------|
| ... | ... | ... |
| вҲ’2 | diminished | d |
| вҲ’1 | minor | m |
| 0 | major | M |
| +1 | augmented | A |
| +2 | doubly augmented | AA |
| ... | ... | ... |

The pattern extends indefinitely in both directions.

**Theorem 3.3.1**: Quality assignment is a total function on **Z** Г— **Z**. Every pair (*c*, *d*) has a unique quality. Conversely, given a quality and a generic interval *d*, the chromatic component *c* is uniquely determined.

*Proof*: Оҙ is a total function on **Z**, deviation is the difference of two integers, and the quality tables cover all integers without overlap. The converse follows because deviation is a bijection between quality labels and integers for each family. вҲҺ

### 3.4 Interval Operations

**Definition 3.4.1** [C]. *Interval addition*: (*c*вӮҒ, *d*вӮҒ) + (*c*вӮӮ, *d*вӮӮ) = (*c*вӮҒ + *c*вӮӮ, *d*вӮҒ + *d*вӮӮ).

**Definition 3.4.2** [C]. *Interval inversion* (within an octave): The inversion of simple interval (*c*, *d*) is (12 вҲ’ *c*, 7 вҲ’ *d*). This gives the complementary interval: a major third (4, 2) inverts to a minor sixth (8, 5); a perfect fifth (7, 4) inverts to a perfect fourth (5, 3).

**Invariant**: For any simple interval *i*, *i* + invert(*i*) = (12, 7), the octave.

**Definition 3.4.3** [C]. *Interval from spelled pitches*: Given spelled pitches *p*вӮҒ = (*l*вӮҒ, *a*вӮҒ, *o*вӮҒ) and *p*вӮӮ = (*l*вӮӮ, *a*вӮӮ, *o*вӮӮ):

- *d* = (*l*вӮӮ вҲ’ *l*вӮҒ) + 7 В· (*o*вӮӮ вҲ’ *o*вӮҒ)  (accounting for letter-name wraparound)
- *c* = *midi*(*p*вӮӮ) вҲ’ *midi*(*p*вӮҒ)

The result (*c*, *d*) is a fully qualified diatonic interval.

**Definition 3.4.4** [C]. *Applying an interval to a spelled pitch*: Given spelled pitch *p* = (*l*, *a*, *o*) and interval (*c*, *d*):

- *l*вҖІ = (*l* + *d*) mod 7
- The octave adjustment accounts for letter wraparound: *o*вҖІ = *o* + вҢҠ(*l* + *d*) / 7вҢӢ (adjusting for the fact that the diatonic scale wraps between B and C)
- *a*вҖІ = (*nat*(*l*) + *a* + *c*) вҲ’ *nat*(*l*вҖІ) вҲ’ 12 В· (*o*вҖІ вҲ’ *o* вҲ’ ...) вҖ” the accidental is whatever is needed to make the chromatic component exact.

More precisely: *a*вҖІ = *c* вҲ’ (Оҙ(*d*) вҲ’ Оҙ(0)) + *a* ... this requires care. Let us define it constructively:

*midi*(*p*вҖІ) = *midi*(*p*) + *c*

*l*вҖІ = (*l* + *d*) mod 7

*o*вҖІ = *o* + вҢҠ(*l* + *d* + (*o* + 1) В· 7 вҲ’ (*o* + 1) В· 7) / 7вҢӢ вҖ” simplifying: the octave increments are determined by how many times the letter index crosses the BвҶ’C boundary.

*a*вҖІ = *midi*(*p*вҖІ) вҲ’ 12 В· (*o*вҖІ + 1) вҲ’ *nat*(*l*вҖІ)

This formulation is constructive and exact. Implementation should compute *midi*(*p*вҖІ) first, then recover (*l*вҖІ, *a*вҖІ, *o*вҖІ) from the known diatonic target and the required MIDI pitch.

**Status**: Implemented. PTDI001A: interval quality derivation, interval inversion, and interval application on SpelledPitch.

### 3.5 Interval Class Vector

**Definition 3.5.1** [C]. The *interval class vector* (ICV) of a pitch class set *S* = {*s*вӮҒ, *s*вӮӮ, ..., *s*вӮ–} is the 6-tuple вҹЁ*v*вӮҒ, *v*вӮӮ, *v*вӮғ, *v*вӮ„, *v*вӮ…, *v*вӮҶвҹ© where *v*бөў counts the number of unordered pairs {*a*, *b*} вҠӮ *S* with *ic*(*a*, *b*) = *i*.

**Properties**:
- The total count is вҲ‘бөў *v*бөў = *k*(*k* вҲ’ 1) / 2.
- Two pitch class sets with the same ICV but different prime forms are *Z-related*.

**Status**: Implemented.

## 4. Scale and Mode Theory

### 4.1 Scale as Pitch Class Set

**Definition 4.1.1** [C]. A *scale type* is an ordered subset of **Z/12Z**, represented as a binary vector *s* вҲҲ {0, 1}В№ВІ where *s*[*i*] = 1 iff pitch class *i* is a member of the scale rooted at pitch class 0. The *cardinality* of the scale is |*s*| = вҲ‘бөў *s*[*i*].

**Definition 4.1.2** [C]. A *scale instance* is a pair (*s*, *r*) where *s* is a scale type and *r* вҲҲ **Z/12Z** is the root. The pitch class set of the instance is {(*r* + *i*) mod 12 : *s*[*i*] = 1}.

**Definition 4.1.3** [C]. The *interval pattern* of a scale type *s* of cardinality *k* is the sequence (*i*вӮҒ, *i*вӮӮ, ..., *i*вӮ–) of successive intervals between consecutive scale degrees (including the wraparound interval from the last degree back to the root plus octave), where вҲ‘вұј *i*вұј = 12.

### 4.2 Scale Catalogue

### 4.4 Scale Relationships

**Definition 4.4.1** [C]. Two scale instances (*s*вӮҒ, *r*вӮҒ) and (*s*вӮӮ, *r*вӮӮ) are:

### 4.6 Generated Scales

**Definition 4.6.1** [C]. A *generated scale* is a pitch class set obtained by iterating a fixed interval *g* (the generator) from a starting pitch class and collecting *k* distinct pitch classes:

## 5. Chord Construction and Classification

### 5.1 Chord as Pitch Class Set with Root

**Definition 5.1.1** [C]. A *chord type* is a pair (*R*, *S*) where *R* is the root pitch class (or the interval 0, for abstract types) and *S* вҠӮ **Z/12Z** is a set of intervals above the root.

A *chord instance* is a chord type with a specified root *r* вҲҲ **Z/12Z**: the pitch class set is {(*r* + *s*) mod 12 : *s* вҲҲ *S*}.

**Distinction from unstructured pitch class set**: A chord has a designated root, which determines function, inversion, and naming. The pitch class set {0, 4, 7} with root 0 is C major; with root 4 is E minor first inversion (depending on voicing).

### 5.4 Non-Tertian Chord Construction

**Definition 5.4.1** [C]. *Quartal chords* are constructed by stacking perfect fourths (5 semitones):

- Quartal triad: {0, 5, 10}
- Quartal tetrachord: {0, 5, 10, 3}

**Definition 5.4.2** [C]. *Quintal chords* are constructed by stacking perfect fifths (7 semitones). Since the fifth is the inversion of the fourth, quintal voicings are inversions of quartal voicings.

**Definition 5.4.3** [C]. *Secundal clusters* are constructed from stacked seconds (1 or 2 semitones). A *tone cluster* of width *w* semitones starting at root *r* is the pitch class set {*r*, *r*+1, *r*+2, ..., *r*+*w*} mod 12.

**Definition 5.4.4** [C]. *Polychords* are the superposition of two or more triads. A polychord is represented as an ordered pair (or tuple) of chord types:

- Upper structure triad over a lower chord: e.g., D major over C7 yields C13(вҷҜ11).
- The pitch class set is the union; the harmonic function depends on the lower chord's root.

### 5.7 Chord Root Finding

**Definition 5.7.1** [H]. Given an unordered pitch class set *S*, the *root finding problem* is to determine the most likely root pitch class and chord quality.

**Algorithm** (tertian stacking heuristic, currently implemented):

For each candidate root *r* вҲҲ *S*:
1. Compute intervals above *r* for all members of *S*.
2. Score the intervals by how well they fit a tertian stacking pattern.
3. The candidate with the highest score is the root; the interval configuration determines quality.

This is heuristic because:
- Multiple roots may be plausible (e.g., Am7 and C6 share the same pitch class set).
- Inversion and voicing affect perceived root.
- Context (preceding and following chords) disambiguates.

**Status**: Implemented (tertian stacking heuristic). Contextual disambiguation is absent.

### 6.5 Harmonic Progression as State Transformation

**Definition 6.5.1** [C]. A *harmonic state* is a tuple (*K*, *C*, *F*, *H*) where:

- *K* is the current key (scale type + root)
- *C* is the current chord (pitch class set + root + quality)

- *H* is the history (sequence of preceding states)

**Definition 6.5.2** [C]. A *harmonic transition* is a function П„ : HarmonicState вҶ’ HarmonicState that transforms the current state by selecting a new chord. Transitions are classified as:

- *Chromatic*: New chord is non-diatonic and non-functional (e.g., chromatic planing, common-tone motion).

## 7. Voice Leading and Counterpoint

### 7.1 Voice Leading as Mapping

**Definition 7.1.1** [C]. A *voice leading* from chord *A* = {*a*вӮҒ, ..., *a*вӮ–} to chord *B* = {*b*вӮҒ, ..., *b*вӮ–} (equal cardinality) is a bijection Пғ : *A* вҶ’ *B* together with a *displacement function* Оҙ : *A* вҶ’ **Z** such that *b*_{Пғ(i)} = *a*_i + Оҙ(*a*_i), where Оҙ(*a*_i) is the signed semitone displacement of voice *i*.

The *total voice-leading distance* is вҲ‘бөў |Оҙ(*a*бөў)|.

When the chords have unequal cardinality, voice leading is a partial function or involves voice splitting/merging, which the specification treats as a constraint relaxation (В§7.5).

### 7.2 Parsimonious Voice Leading

**Definition 7.2.1** [C]. A voice leading is *parsimonious* if the total voice-leading distance is minimised over all possible bijections Пғ. This is an instance of the assignment problem, solvable in O(*k*Ві) by the Hungarian algorithm or, for small *k* (typical in music: *k* вүӨ 8), by exhaustive search.

**Definition 7.2.2** [C]. The *nearest-tone algorithm* (greedy approximation): Assign each voice in *A* to its nearest unassigned pitch in *B*. This is O(*k*ВІ) and produces optimal or near-optimal results for typical chord sizes.

**Status**: Implemented. VLNT001A: nearest-tone greedy algorithm (`voice_lead_nearest_tone`) and O(kВі) Hungarian algorithm (`voice_lead_optimal`).

### 7.3 Motion Classification

**Definition 7.3.1** [C]. Given two voices moving from (*a*вӮҒ, *a*вӮӮ) to (*b*вӮҒ, *b*вӮӮ), the *motion type* is:

| Condition | Motion |
|-----------|--------|
| *b*вӮҒ вҲ’ *a*вӮҒ = *b*вӮӮ вҲ’ *a*вӮӮ (same displacement) | Parallel |
| sign(*b*вӮҒ вҲ’ *a*вӮҒ) = sign(*b*вӮӮ вҲ’ *a*вӮӮ) but different magnitude | Similar |
| sign(*b*вӮҒ вҲ’ *a*вӮҒ) вү  sign(*b*вӮӮ вҲ’ *a*вӮӮ) | Contrary |
| One voice stationary, the other moves | Oblique |

**Status**: Implemented. General motion classification (`classify_voice_motion`) returns Parallel, Similar, Contrary, Oblique, or Static. Parallel motion detection for fifths/octaves also available.

### 7.4 Common-Practice Voice-Leading Constraints

**Definition 7.4.1** [H]. The following constraints derive from common-practice part-writing. Each constraint has a severity (error, warning, preference) configurable by style:

| Rule | Description | Severity |
|------|------------|----------|

### 7.6 Species Counterpoint

**Definition 7.6.1** [H]. *Species counterpoint* (after Fux) is a pedagogical system of constraints for composing a counterpoint voice against a given *cantus firmus*. The five species are:

**First species** (note against note): Each note of the counterpoint aligns with one note of the cantus firmus.

Constraints:
- Begin and end on a perfect consonance (P1, P5, P8).
- All intervals between the two voices must be consonant (P1, m3, M3, P5, m6, M6, P8).
- No parallel P5s or P8s.
- No direct (hidden) P5s or P8s.
- Contrary and oblique motion preferred.
- No voice crossing.
- Approach the final note by step (leading tone from below, or step from above).

**Second species** (two notes against one): Two counterpoint notes per cantus firmus note. The first note (strong beat) follows first-species consonance rules. The second note (weak beat) may be a passing tone (consonant or dissonant by step).

**Third species** (four notes against one): Four counterpoint notes per cantus firmus note. Consonance required on the strong beat; passing tones and neighbour tones permitted on weak beats.

**Fourth species** (syncopation): The counterpoint is displaced by a half-note, creating suspensions. A dissonant suspension on the strong beat must resolve down by step to a consonance.

**Fifth species** (florid): Combines elements of all four preceding species.

The specification provides these as constraint sets; an implementation may solve them as constraint satisfaction problems or use them to validate existing counterpoint.

**Status**: Implemented. VLSC001A: validation for all five species (`check_first_species` through `check_fifth_species`, `check_species` dispatcher) plus first-species backtracking solver (`solve_first_species`).

### 7.7 Figured Bass Realisation

**Definition 7.7.1** [H]. *Figured bass* is a notational system in which a bass line is annotated with numbers indicating intervals above the bass. The *realisation problem* is: given a bass note and figured bass symbols, determine a complete voicing satisfying the indicated intervals and the voice-leading constraints of В§7.4.

| 7 | 3rd, 5th, 7th | Root position 7th |
| 6/5 | 3rd, 5th, 6th | First inversion 7th |
| 4/3 | 3rd, 4th, 6th | Second inversion 7th |
| 4/2 or 2 | 2nd, 4th, 6th | Third inversion 7th |

Accidentals in figures: A slash through a number, or вҷҜ/вҷӯ before a number, raises/lowers the indicated interval. A standalone accidental applies to the 3rd.

## 8. Melody

### 8.1 Melodic Contour

**Definition 8.1.1** [C]. The *contour* of a melody is the sequence of directions between successive pitches. For a pitch sequence (*p*вӮҒ, *p*вӮӮ, ..., *p*вӮҷ), the contour is the sequence (*c*вӮҒ, *c*вӮӮ, ..., *c*вӮҷвӮӢвӮҒ) where:

- *c*бөў = +1 if *p*бөўвӮҠвӮҒ > *p*бөў (ascending)
- *c*бөў = 0 if *p*бөўвӮҠвӮҒ = *p*бөў (stationary)
- *c*бөў = вҲ’1 if *p*бөўвӮҠвӮҒ < *p*бөў (descending)

**Definition 8.1.2** [C]. *Contour reduction* (after Morris): Reduce a pitch sequence to its boundary pitches (local maxima and minima) to extract a simplified contour representation. This is recursive: reduce, then reduce again, until a fixed point.

### 8.2 Melodic Motion Classification

**Definition 8.2.1** [C]. An interval between successive melody notes is:

- *Conjunct* (step) if |interval| вүӨ 2 semitones.
- *Disjunct* (leap) if |interval| > 2 semitones.

A melody is *predominantly conjunct* if the proportion of conjunct intervals exceeds a threshold (typically > 0.6, configurable).

### 8.3 Melodic Statistics

**Definition 8.3.1** [C]. For a pitch sequence (*p*вӮҒ, ..., *p*вӮҷ):

- *Range*: max(*pбөў*) вҲ’ min(*pбөў*)
- *Tessitura*: The pitch interval containing the central mass of the melody (e.g., the interval from the 10th to the 90th percentile of pitch values)
- *Mean pitch*: (1/*n*) В· вҲ‘бөў *pбөў*
- *Interval histogram*: Frequency distribution of directed intervals between successive notes
- *Pitch class histogram*: Frequency distribution of pitch classes

### 8.4 Melodic Tendency Tones

**Definition 8.4.1** [H]. Certain scale degrees carry *tendency* вҖ” a stylistic expectation of resolution direction:

| Degree | Tendency | Resolution |
|--------|----------|------------|
| 7МӮ (leading tone) | Upward | вҶ’ 1МӮ |
| 4МӮ | Downward | вҶ’ 3МӮ |
| вҷӯ6МӮ (in minor) | Downward | вҶ’ 5МӮ |
| вҷҜ4МӮ (raised) | Upward | вҶ’ 5МӮ |
| Chromatic alterations | Toward direction of alteration | вҷҜвҶ’up, вҷӯвҶ’down |

These are heuristic preferences from common-practice style, not absolute rules.

### 8.5 Sequence Detection

**Definition 8.5.1** [C]. A *real sequence* is a repetition of a melodic pattern transposed by a fixed chromatic interval. A *tonal sequence* (or *diatonic sequence*) is a repetition of a melodic pattern transposed by a fixed diatonic interval (the chromatic intervals vary to stay within the key).

**Detection algorithm** [C]: Given a melody and a minimum pattern length *l*:
1. For each starting position *i*, extract the interval pattern of length *l*.
2. Search for subsequent occurrences of the same (or diatonically equivalent) interval pattern.
3. Report matches with their transposition level and type (real or tonal).

## 9. Rhythm and Metre

### 9.1 Beat as Exact Rational

**Definition 9.1.1** [C]. A *beat value* is an element of **Q+** (positive rationals), represented as an irreducible fraction *p*/*q* where *p*, *q* вҲҲ **Z+** and gcd(*p*, *q*) = 1.

Arithmetic operations on beats use exact rational arithmetic:
- Addition: *a*/*b* + *c*/*d* = (*ad* + *bc*) / *bd*, reduced.
- Multiplication: (*a*/*b*) В· (*c*/*d*) = *ac* / *bd*, reduced.
- Comparison: *a*/*b* < *c*/*d* iff *ad* < *bc*.

**Invariant**: No floating-point representation is used in any rhythmic calculation. All rhythmic positions and durations are exactly representable.

**Status**: Implemented (Beat type with exact rational arithmetic).

### 9.2 Time Signature

**Definition 9.2.1** [C]. A *time signature* is a pair (*numerator*, *denominator*) specifying the number of beats per measure and the note value that receives one beat.

**Definition 9.2.2** [C]. Time signatures are classified by their *metre type*:

| Type | Condition | Beat Division | Examples |
|------|-----------|--------------|---------|
| Simple | Numerator = 2, 3, or 4; beats divide into 2 | Duple (binary) | 2/4, 3/4, 4/4 |
| Compound | Numerator = 6, 9, or 12; beats divide into 3 | Triple (ternary) | 6/8, 9/8, 12/8 |
| Asymmetric (additive) | Numerator not in {2,3,4,6,9,12} | Mixed | 5/8, 7/8, 11/8 |
| Complex | Nested groupings | Hierarchical | 3+3+2/8 |

**Definition 9.2.3** [C]. An *additive metre* is specified as a partition of the numerator: e.g., 7/8 as (2+2+3) or (3+2+2). The partition determines the beat grouping and the accent pattern.

**Representation**: A time signature is stored as a tuple (*groups*, *denominator*) where *groups* is a list of integers summing to the numerator. For simple and compound metres, this is a list of identical values (e.g., 4/4 = (1, 1, 1, 1)/4 or equivalently (4)/4).

### 9.3 Metrical Hierarchy

**Definition 9.3.1** [C]. A *metrical hierarchy* assigns a weight (accent level) to each metrical position. For a measure of *N* pulses, the hierarchy is defined recursively:

- The downbeat (position 0) has the highest weight.
- In simple metre, positions at powers of 2 subdivisions have weight proportional to the subdivision level.
- In compound metre, the ternary beat grouping adds an additional level.

Formally, for a measure with *N* pulses, the *metrical weight* of position *k* is:

*w*(*k*, *N*) = max{*j* вҲҲ **ZвүҘ0** : *k* mod (*N* / 2КІ) = 0}  (for simple binary metres)

For ternary subdivisions, the recursion branches by 3 instead of 2 at the appropriate level.

**Definition 9.3.2** [C]. *Syncopation* is rhythmic emphasis that contradicts the prevailing metrical hierarchy. A note onset at metrical position *k* is syncopated if:
- *w*(*k*) < *w*(*k* + *d*), where *d* is the duration of the note, meaning the note sustains through a metrically stronger position than its onset.

### 9.4 Tuplets

**Definition 9.4.1** [C]. A *tuplet* divides a beat span into a number of equal parts different from the normal subdivision. An *m*:*n* tuplet places *m* notes in the span normally occupied by *n* notes of the same written value.

| Tuplet | Ratio | Example |
|--------|-------|---------|
| Triplet | 3:2 | 3 eighth notes in the span of 2 |
| Duplet (in compound) | 2:3 | 2 notes in the span of 3 |
| Quintuplet | 5:4 | 5 notes in the span of 4 |
| Septuplet | 7:4 | 7 notes in the span of 4 |

The duration of each note in an *m*:*n* tuplet is (*n*/*m*) times the normal duration of that note value.

Since durations are represented as exact rationals (В§9.1), tuplet durations are exactly representable: a triplet eighth note has duration (2/3) В· (1/2 beat) = 1/3 beat.

**Nested tuplets**: A tuplet within a tuplet produces a compound ratio. A triplet of quintuplets: each sub-note has duration (2/3) В· (4/5) В· *d* = (8/15) В· *d*, where *d* is the base note duration. Exact rational arithmetic handles this without approximation.

### 9.5 Polyrhythm and Polymetre

**Definition 9.5.1** [C]. A *polyrhythm* is the simultaneous sounding of two or more rhythmic patterns whose pulse counts within the same span are co-prime (or at least distinct). An *m*-against-*n* polyrhythm places *m* equally-spaced onsets against *n* equally-spaced onsets within the same time span.

The onset times for the *m*-stream are {*k*/*m* : *k* = 0, 1, ..., *m* вҲ’ 1} and for the *n*-stream are {*k*/*n* : *k* = 0, 1, ..., *n* вҲ’ 1}. The combined onset pattern has gcd(*m*, *n*) coincident onsets.

**Definition 9.5.2** [C]. *Polymetre* is the simultaneous use of different time signatures. Unlike polyrhythm (which shares a common span), polymetre has different bar lengths that align only at their LCM.

The LCM alignment point for metres of length *a* beats and *b* beats is lcm(*a*, *b*) beats.

### 9.6 Euclidean Rhythm

**Definition 9.6.1** [C]. An *Euclidean rhythm* E(*k*, *n*) distributes *k* onsets as evenly as possible among *n* time slots, computed by Bjorklund's algorithm (equivalent to the Bresenham line-drawing algorithm).

**Properties**:
- E(*k*, *n*) maximises the sum of inter-onset intervals' minimum values.
- E(*k*, *n*) is a rotation of E(*n* вҲ’ *k*, *n*) (the complement).
- E(*k*, *n*) with rotation *r* is denoted E(*k*, *n*)*r*, where onset positions are cyclically shifted by *r* slots.

**Status**: Implemented with presets (Tresillo, Cinquillo, Son clave, Rumba clave, Bossa nova, Gahu).

### 9.7 Rhythmic Transformations

**Definition 9.7.1** [C]. Given a rhythmic pattern (sequence of durations and onsets):

- *Augmentation*: Multiply all durations by a factor *f* > 1.
- *Diminution*: Multiply all durations by a factor 0 < *f* < 1.
- *Retrograde*: Reverse the order of the duration sequence.
- *Rotation*: Cyclic permutation of onsets within a repeating pattern.

Since durations are exact rationals, these transformations preserve exactness.

### 9.8 Metric Modulation

**Definition 9.8.1** [C]. A *metric modulation* occurs when a subdivision of the old tempo becomes the new beat unit, effecting a tempo change by a rational factor.

If the old tempo is *T*вӮҒ BPM with beat unit *b*вӮҒ, and a *p*:*q* tuplet in the old tempo becomes the new beat, the new tempo is:

*T*вӮӮ = *T*вӮҒ В· (*p*/*q*) В· (*b*вӮҒ / *b*вӮӮ)

where *b*вӮӮ is the new beat unit. Since all values are rational, the tempo ratio is exact.

### 9.9 Swing and Shuffle

**Definition 9.9.1** [H]. *Swing* displaces the second note of each pair of equal subdivisions by a *swing ratio* ПҒ вҲҲ (0.5, 1.0). A straight pair divides the beat 50/50; a swing ratio of ПҒ places the first note at duration ПҒ В· *d* and the second at (1 вҲ’ ПҒ) В· *d*, where *d* is the full beat duration.

Common ratios:
- Straight: ПҒ = 0.5
- Light swing: ПҒ вүҲ 0.58
- Triplet swing: ПҒ = 2/3
- Hard swing: ПҒ вүҲ 0.75

The triplet swing ratio 2/3 is exactly representable as a rational.

## 10. Form and Large-Scale Structure

### 10.1 Formal Units as Recursive Grammar

**Definition 10.1.1** [H]. Musical form is modelled as a recursive grammar over structural units. A *formal unit* is a labelled, bounded region of musical time containing harmonic and thematic content.

**Grammar** (context-free, extended):

```
composition   ::= section+
section       ::= label ":" phrase_group+
phrase_group  ::= phrase+
phrase        ::= subphrase+ cadence
subphrase     ::= motif+
```

where `label` вҲҲ {A, B, C, ..., Introduction, Coda, Bridge, Development, Recapitulation, ...}.

### 10.2 Phrase Structure

**Definition 10.2.1** [H]. A *phrase* is the smallest unit of musical form that terminates with a cadence. Common phrase structures:

**Sentence** (Satz): 1 + 1 + 2 bar grouping.
- *Presentation phrase* (bars 1вҖ“4): basic idea (2 bars) + repetition/variant (2 bars).
- *Continuation phrase* (bars 5вҖ“8): fragmentation, acceleration, cadence.

**Period**: Two balanced phrases.
- *Antecedent* (4 bars): ends with a half cadence.
- *Consequent* (4 bars): ends with a perfect authentic cadence.
- Parallel period: consequent begins like the antecedent.
- Contrasting period: consequent has different opening material.

### 10.3 Sectional Forms

| Form | Structure | Description |
|------|-----------|-------------|
| Binary | A B or A A' B B' | Two sections, often with repeat signs |
| Rounded binary | A B A' | B section returns to A material |
| Ternary | A B A | Three sections; B contrasts with A |
| Rondo | A B A C A (etc.) | Recurring refrain alternates with episodes |
| Sonata | ExpoвҖ“DevвҖ“Recap | Exposition (2 themes), development, recapitulation |
| Theme & variations | A A' A'' A''' ... | Theme followed by varied repetitions |
| Strophic | A A A A ... | Same music, different text (vocal) |
| Through-composed | A B C D ... | No large-scale repetition |

### 10.4 Motivic Analysis

**Definition 10.4.1** [H]. A *motif* is a short melodic-rhythmic figure that serves as a building block for larger structures. Motivic transformations:

| Transformation | Description |
|---------------|-------------|
| Repetition | Exact restatement |
| Transposition | Same intervals, different pitch level |
| Sequence | Repeated transposition (В§8.5) |
| Inversion | Intervals reflected |
| Retrograde | Time-reversed |
| Retrograde-inversion | Both reflected and reversed |

| Diminution | Durations divided |

| Extension | Motif lengthened by addition or repetition |

Motivic analysis is heuristic because motif identity is not a purely formal property; similarity thresholds and contextual relevance require aesthetic judgement.

## 11. Transformational Theory

### 11.1 Neo-Riemannian Operations

**Definition 11.1.1** [C]. The three principal neo-Riemannian operations act on the set of 24 major and minor triads (the consonant triads). Each operation is an involution (self-inverse) and changes exactly one pitch class by a semitone:

**P** (Parallel): Exchanges major and minor triads sharing a root.
- *P*({0, 4, 7}) = {0, 3, 7} (C major вҶ’ C minor)
- Changes the third; root and fifth are common tones.

**R** (Relative): Exchanges relative major and minor triads.
- *R*({0, 4, 7}) = {4, 7, 11} вүЎ {9, 0, 4} (C major вҶ’ A minor)
- Wait вҖ” more carefully: *R*({0, 4, 7}) = {0, 4, 9} (C major вҶ’ A minor).
- Changes the fifth of the major triad (or the root of the minor triad); two common tones.

Let me define these precisely by their action on the consonant triad:

Represent a triad as (root, quality) where quality вҲҲ {major, minor}.

| Operation | (r, major) вҶ’ | (r, minor) вҶ’ |
|-----------|-------------|--------------|
| P | (r, minor) | (r, major) |
| R | ((r + 9) mod 12, minor) | ((r + 3) mod 12, major) |
| L | ((r + 4) mod 12, minor) | ((r + 8) mod 12, major) |

Or equivalently in terms of pitch class sets:

**P**: {*r*, *r*+4, *r*+7} вҶ” {*r*, *r*+3, *r*+7}

**R**: {*r*, *r*+4, *r*+7} вҶ” {*r*+9, *r*, *r*+4} (= A minor if root is C)

**L** (Leading-Tone Exchange): {*r*, *r*+4, *r*+7} вҶ” {*r*+4, *r*+7, *r*+11} (= E minor if root is C)

**Verification of involution**: *P*ВІ = *R*ВІ = *L*ВІ = identity. Each exchanges two triads and maps every other triad to a definite target, so applying twice returns to the original.

### 11.2 The PLR Group

**Definition 11.2.1** [C]. The group generated by P, L, R under composition is the *PLR group*, isomorphic to the dihedral group of order 24 acting on the set of 24 consonant triads. This group is also known as the *Schritt-Wechsel group*.

Compound operations of musical interest:

| Compound | = | Musical Effect |
|----------|---|---------------|
| LP | L вҲҳ P | Slide transformation (e.g., C major вҶ’ CвҷҜ minor) |
| RP | R вҲҳ P | Nebenverwandt (e.g., C major вҶ’ F minor) |
| PLP | P вҲҳ L вҲҳ P | Hexatonic pole (e.g., C major вҶ’ Aвҷӯ major) вҖ” maximally distant in the hexatonic cycle |
| LPL = RPR | | Equivalent to transposition by tritone with quality change |

### 11.3 The Tonnetz

**Definition 11.3.1** [C]. The *Tonnetz* is a two-dimensional lattice in which:

- The horizontal axis represents major thirds (interval 4).
- The vertical axis represents minor thirds (interval 3).
- Equivalently: one axis is perfect fifths (7), the other is major thirds (4).

Under 12-TET, the Tonnetz tiles the torus **Z/12Z** Г— **Z/12Z** (since both axis generators eventually cycle). Each triangular cell of the Tonnetz corresponds to a consonant triad.

The neo-Riemannian operations correspond to specific geometric moves on the Tonnetz:
- **P**: Reflects across the edge shared by the two triangles of a (root, third) edge.
- **R**: Reflects across the edge shared by root and fifth.
- **L**: Reflects across the edge shared by third and fifth.

**Distance on the Tonnetz**: The minimum number of PLR moves between two triads provides a measure of their transformational distance.

### 11.4 Generalised Interval System (GIS)

**Definition 11.4.1** [C]. A *Generalised Interval System* (Lewin) is a triple (*S*, IVLS, int) where:

- *S* is a set of musical objects.
- IVLS is a group of intervals.
- int : *S* Г— *S* вҶ’ IVLS is a function satisfying:
  - For all *s*, *t*, *u* вҲҲ *S*: int(*s*, *t*) В· int(*t*, *u*) = int(*s*, *u*)
  - For each *s* вҲҲ *S* and *i* вҲҲ IVLS, there exists a unique *t* вҲҲ *S* such that int(*s*, *t*) = *i*.

The second condition makes the action of IVLS on *S* *simply transitive* (free and transitive).

**Examples of GIS instances**:

| *S* | IVLS | int |
|-----|------|-----|
| Pitch classes (**Z/12Z**) | (**Z/12Z**, +) | int(*s*, *t*) = *t* вҲ’ *s* mod 12 |
| Pitches (**Z**) | (**Z**, +) | int(*s*, *t*) = *t* вҲ’ *s* |
| Time points (**Q**) | (**Q**, +) | int(*s*, *t*) = *t* вҲ’ *s* |
| Consonant triads | Schritt-Wechsel group | neo-Riemannian transformation |

### 11.5 Klumpenhouwer Networks

**Definition 11.5.1** [H]. A *Klumpenhouwer network* (K-net) is a directed graph where:

- Nodes are pitch classes (or pitch class sets).
- Edges are labelled with transformations from the T/I group (transposition or inversion operations).
- The graph is *well-formed* if, for every cycle, the composition of edge labels equals the identity.

K-nets are used to express structural relationships between pitch collections that are not related by simple transposition or inversion.

The identification and comparison of K-nets involves the notion of *isography* (same graph structure, possibly different labels) and *strong isography* (same graph with labels related by a uniform transformation), which are heuristic analytical judgements.

## 12. Post-Tonal and Set-Theoretic Methods

### 12.1 Pitch Class Sets

**Definition 12.1.1** [C]. A *pitch class set* (pcset) is a subset of **Z/12Z**. Its *cardinality* is the number of elements; standard terminology: dyad (2), trichord (3), tetrachord (4), pentachord (5), hexachord (6), heptachord (7), octachord (8), nonachord (9).

### 12.2 Normal Form

**Definition 12.2.1** [C]. The *normal form* of a pcset *S* is the rotation of *S* (listed in ascending pitch class order) that is "most packed to the left." Algorithm:

1. List the elements of *S* in ascending order.
2. Consider all *k* rotations (where *k* = |*S*|).
3. For each rotation, compute the interval from first to last element.
4. Select the rotation with the smallest such interval.
5. Ties broken by comparing successive intervals from the left (smallest first wins).

**Status**: Implemented.

### 12.3 Prime Form

**Definition 12.3.1** [C]. The *prime form* of a pcset *S* is the representative of the TI-equivalence class of *S* that begins with 0 and is most packed to the left. Algorithm:

1. Compute normal form of *S* вҶ’ *N*.
2. Compute normal form of the inversion of *S* (i.e., {12 вҲ’ *s* : *s* вҲҲ *S*}) вҶ’ *N*вҖІ.
3. Transpose both *N* and *N*вҖІ to start on 0.
4. Select whichever is more packed to the left.

**Status**: Implemented.

### 12.4 Forte Number Catalogue

**Definition 12.4.1** [C]. The *Forte number* is a standard label for each set class (TI-equivalence class), written as *c*-*n* where *c* is the cardinality and *n* is the ordinal position in Forte's catalogue.

The complete catalogue contains 220 distinct set classes (including the empty set, singletons, and the aggregate). The specification includes the full catalogue as a lookup table mapping prime form вҶ’ Forte number.

### 12.5 Z-Relation

**Definition 12.5.1** [C]. Two pcsets are *Z-related* if they share the same interval class vector but are not related by transposition or inversion (i.e., they have different prime forms).

There are 19 Z-related pairs in the 12-TET system.

### 12.6 Similarity Relations

**Definition 12.6.1** [C]. Several similarity measures between pcsets:

- **RвӮҖ**: *A* and *B* have the same cardinality, and their ICVs differ in every entry (maximum dissimilarity).
- **RвӮҒ**: *A* and *B* have the same cardinality, and their ICVs differ in exactly one entry, by exactly 1.
- **RвӮӮ**: *A* and *B* have the same cardinality, and their ICVs differ in exactly one entry (by any amount).
- **RвӮҡ**: *A* and *B* have the same cardinality, and one can be embedded in the other after at most one transposition (the "inclusion" relation).

### 12.7 Twelve-Tone Serialism

**Definition 12.7.1** [C]. A *tone row* (series) is an ordered sequence of all 12 pitch classes, each occurring exactly once: a permutation of **Z/12Z**.

A row *P*вӮҖ = (*p*вӮҖ, *p*вӮҒ, ..., *p*вӮҒвӮҒ) generates a family of 48 row forms:

| Operation | Notation | Definition |
|-----------|----------|-----------|
| Transposition | *P*вӮҷ | (*p*вӮҖ+*n*, *p*вӮҒ+*n*, ..., *p*вӮҒвӮҒ+*n*) mod 12 |
| Inversion | *I*вӮҷ | (*n*вҲ’*p*вӮҖ, *n*вҲ’*p*вӮҒ, ..., *n*вҲ’*p*вӮҒвӮҒ) mod 12 |
| Retrograde | *R*вӮҷ | reverse of *P*вӮҷ |
| Retrograde-Inversion | *RI*вӮҷ | reverse of *I*вӮҷ |

These are computed for all *n* вҲҲ {0, 1, ..., 11}, yielding (at most) 48 distinct row forms.

### 12.8 Twelve-Tone Matrix

**Definition 12.8.1** [C]. The *twelve-tone matrix* (or *magic square*) for row *P*вӮҖ is a 12Г—12 matrix *M* where:

- Row *i* = *P*вӮҷ where *n* = *I*вӮҖ[*i*] (i.e., transposition such that the first element of row *i* is *I*вӮҖ[*i*]).
- Column *j* = *I*вӮҷ where *n* = *P*вӮҖ[*j*].
- Reading any row left-to-right gives a prime form.
- Reading any column top-to-bottom gives an inversion form.
- Reading any row right-to-left gives a retrograde form.
- Reading any column bottom-to-top gives a retrograde-inversion form.

### 12.9 Combinatoriality

**Definition 12.9.1** [C]. A hexachord (6-element subset of a row) is *combinatorial* if its complement (the other 6 pitch classes) can be produced by one of the four row operations applied to the hexachord itself.

Types of combinatoriality:
- *P-combinatorial*: A transposition of the hexachord yields its complement.
- *I-combinatorial*: An inversion yields the complement.
- *R-combinatorial*: The retrograde yields the complement (trivially true for all hexachords).
- *RI-combinatorial*: A retrograde-inversion yields the complement.
- *All-combinatorial*: Satisfies all four types (there are exactly 6 all-combinatorial hexachords).

## 13. Tuning and Temperament

### 13.1 Equal Temperament Generalisation

**Definition 13.1.1** [C]. An *n*-tone equal temperament (*n*-TET or *n*-EDO) divides the octave into *n* equal steps, each of frequency ratio 2^(1/*n*). The step size in cents is 1200/*n*.

The pitch class group generalises to **Z**/*n***Z** for *n*-EDO. All operations defined over **Z/12Z** in this specification generalise by replacing 12 with *n* wherever it appears as a modulus.

**Mapping to frequency**: For pitch *p* in *n*-EDO:

*f*(*p*) = *f*_ref В· 2^(*p* / *n*)

**Common EDO systems**:

| *n* | Step (cents) | Approximation Quality |
|-----|-------------|----------------------|
| 12 | 100.0 | Standard Western |
| 19 | 63.16 | Better major thirds |
| 24 | 50.0 | Quarter-tones |
| 31 | 38.71 | Excellent thirds and fifths |
| 53 | 22.64 | Near-just in 5-limit |

### 13.2 Just Intonation

**Definition 13.2.1** [C]. In *just intonation* (JI), intervals are defined by integer frequency ratios. The *p*-limit of a JI system is the largest prime factor appearing in any ratio.

**5-limit intervals** (involving primes 2, 3, 5):

| Interval | Ratio | Cents |
|----------|-------|-------|
| Unison | 1/1 | 0.0 |
| Minor second | 16/15 | 111.7 |
| Major second | 9/8 | 203.9 |
| Minor third | 6/5 | 315.6 |
| Major third | 5/4 | 386.3 |
| Perfect fourth | 4/3 | 498.0 |
| Tritone | 45/32 | 590.2 |
| Perfect fifth | 3/2 | 702.0 |
| Minor sixth | 8/5 | 813.7 |
| Major sixth | 5/3 | 884.4 |
| Minor seventh | 9/5 | 1017.6 |
| Major seventh | 15/8 | 1088.3 |
| Octave | 2/1 | 1200.0 |

**Definition 13.2.2** [C]. A *comma* is a small interval arising from the difference between two paths to nominally the same pitch. Important commas:

| Comma | Ratio | Cents | Origin |
|-------|-------|-------|--------|
| Syntonic comma | 81/80 | 21.5 | 4 fifths vs. 1 major third |
| Pythagorean comma | 531441/524288 | 23.5 | 12 fifths vs. 7 octaves |
| Diesis | 128/125 | 41.1 | 3 major thirds vs. 1 octave |
| Schisma | 32805/32768 | 2.0 | 8 fifths + 1 major third vs. 5 octaves |

### 13.3 Historical Temperaments

**Definition 13.3.1** [H]. A *temperament* is a tuning system in which some or all just intervals are slightly adjusted (tempered) to achieve certain desirable properties (e.g., closure of the circle of fifths, acceptable consonance in multiple keys).

| Temperament | Principle |
|-------------|-----------|
| Pythagorean | All fifths are pure (3/2) except one "wolf fifth" |
| Quarter-comma meantone | Fifths tempered narrow by 1/4 syntonic comma; pure major thirds |
| Third-comma meantone | Fifths tempered narrow by 1/3 syntonic comma; pure minor thirds |
| Werckmeister III | 4 fifths tempered by 1/4 Pythagorean comma; rest pure |
| Kirnberger III | Progressive tempering from C around the circle of fifths |
| Vallotti | 6 fifths tempered by 1/6 Pythagorean comma |
| Young #2 | Similar to Vallotti with different starting point |
| 12-TET | All fifths equally tempered by 1/12 Pythagorean comma |

Each temperament is specified as a table of 12 pitch deviations from 12-TET, measured in cents. This table is the *tuning table*.

**Definition 13.3.2** [C]. A *tuning table* for a 12-note temperament is a function П„ : **Z/12Z** вҶ’ **R** giving the deviation in cents from the corresponding 12-TET pitch. For 12-TET itself, П„(*p*) = 0 for all *p*.

The frequency for pitch class *p* in octave *o* under tuning table П„ is:

*f*(*p*, *o*) = *f*_ref В· 2^((12 В· *o* + *p* вҲ’ 69 + П„(*p*)/1200))

### 13.4 Scala File Support

**Definition 13.4.1** [C]. The Scala .scl file format specifies a tuning as a list of *n* вҲ’ 1 interval values (in cents or as ratios) defining the steps from the starting pitch to each subsequent degree, with the *n*-th degree implicitly being the octave (1200 cents or 2/1 ratio).

**Parsing**: A .scl file consists of:
1. Description line (comment)
2. Number of notes *n*
3. *n* interval values, each either a decimal number (cents) or a ratio (*p*/*q*)

The parsed result is a tuning table.

### 14.2 Consonance and Dissonance Models

**Definition 14.2.1** [E]. *Sensory consonance* is a psychoacoustic quantity related to the absence of roughness (beating between partials). Two tones are maximally rough when their frequency difference is approximately 1/4 of the critical bandwidth.

**Critical bandwidth** (Bark scale approximation):

CB(*f*) вүҲ 25 + 75 В· (1 + 1.4 В· (*f* / 1000)ВІ)^0.69 Hz

**Plomp-Levelt model** [E]: The dissonance *d* between two pure tones at frequencies *f*вӮҒ < *f*вӮӮ is:

*d*(*f*вӮҒ, *f*вӮӮ) = *e*^(вҲ’*a*В·*s*) вҲ’ *e*^(вҲ’*b*В·*s*)

where *s* = (*f*вӮӮ вҲ’ *f*вӮҒ) / (0.24 В· CB((*f*вӮҒ + *f*вӮӮ)/2)), and *a* вүҲ 3.5, *b* вүҲ 5.75 are empirical constants.

For complex tones (with harmonics), total dissonance is the sum of pairwise dissonances between all partial pairs.

**Sethares model** [E]: Extends Plomp-Levelt to arbitrary spectra, providing a dissonance function over all intervals for a given timbre. The minima of this function correspond to the "consonant" intervals for that timbre вҖ” which for harmonic timbres approximate the just intervals, and for inharmonic timbres produce different consonance patterns.

These models are empirical; parameter values derive from psychoacoustic experiments.

### 14.3 Roughness Calculation

**Definition 14.3.1** [E]. *Roughness* quantifies the amplitude fluctuation perceived when two close frequencies interact. For tones with fundamental frequencies *f*вӮҒ, *f*вӮӮ and partials at amplitudes *a*вӮҷ, *b*вӮҳ:

*R* = вҲ‘вӮҷ вҲ‘вӮҳ *g*(*a*вӮҷ, *b*вӮҳ) В· *d*(*n* В· *f*вӮҒ, *m* В· *f*вӮӮ)

where *g*(*a*, *b*) is an amplitude weighting (e.g., min(*a*, *b*) or *a* В· *b*).

### 14.4 Virtual Pitch and Missing Fundamental

**Definition 14.4.1** [E]. *Virtual pitch* is the phenomenon whereby a listener perceives a fundamental frequency that is not physically present, inferred from the harmonic relationships among the upper partials. This is modelled by finding the fundamental frequency whose harmonic series best matches the observed partials (e.g., Terhardt's algorithm, Goldstein's optimal processor).

This is an empirical model; the specification defines the interface (input: set of partial frequencies; output: estimated fundamental) but delegates the model to a configurable implementation.

## 16. Cross-Domain Invariants and Consistency

### 16.1 Round-Trip Consistency [C]

The following round-trip operations must produce the identity:

1. **Pitch**: frequency вҶ’ MIDI pitch вҶ’ frequency (within floating-point tolerance < 2вҒ»вҒҙвҒ° semitones).
2. **Spelled pitch**: SPN string вҶ’ SpelledPitch вҶ’ SPN string (exact).
3. **Interval**: SpelledPitch pair вҶ’ Interval вҶ’ apply to first pitch вҶ’ recover second pitch (exact).
4. **Scale**: interval pattern вҶ’ binary vector вҶ’ interval pattern (exact).
5. **Chord**: root + quality вҶ’ pitch class set вҶ’ root finding вҶ’ original root + quality (for unambiguous chord types).
6. **Roman numeral**: string вҶ’ chord in key вҶ’ string (for the canonical romanisation).
7. **Tone row**: PвӮҖ вҶ’ twelve-tone matrix вҶ’ read off PвӮҖ from first row (exact).
8. **Tuning**: Scala file вҶ’ TuningTable вҶ’ Scala file (within formatting tolerance).

### 16.2 Conservation Laws [C]

1. **Pitch class cardinality under DвӮҒвӮӮ**: |*g*(*S*)| = |*S*| for all *g* вҲҲ DвӮҒвӮӮ and all pcsets *S*.
2. **ICV under transposition**: ICV(*T*вӮ–(*S*)) = ICV(*S*) for all *k*, *S*.
3. **ICV under inversion**: ICV(*I*вӮ–(*S*)) = ICV(*S*) for all *k*, *S*.
4. **Interval sum in scale**: The sum of the interval pattern of any scale equals *n* (the EDO division).
5. **Tone row bijectivity**: Every row form is a permutation of **Z/12Z**.
6. **Rational arithmetic closure**: All rhythmic operations on **Q** produce results in **Q** (no irrational numbers).
7. **Negative harmony involution**: *neg* вҲҳ *neg* = id.
8. **Neo-Riemannian involution**: *P*ВІ = *R*ВІ = *L*ВІ = id.
